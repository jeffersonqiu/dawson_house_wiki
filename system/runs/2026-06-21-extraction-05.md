1. **7 separate tile records, not one "Hafary tile shortlist" blob.** Each tile is a distinct
   shortlist candidate for a specific room/zone. Keeping them separate means the Compiler can
   compile only the ones the human approves per room, and future status updates (ordered/rejected)
   are tracked individually.

2. **ST7530K series split between Common Bathroom (K17) and Master Bathroom (K14/K15/K11).**
   The same product family is appearing in two rooms. K17 is the Common Bathroom top shortlist;
   K14/K15/K11 appear to be Master Bathroom shower/soap zone options. Kept separate rather than
   merged — the use-case (common bathroom feature wall vs. master bathroom wet area) differs even
   though the vendor code series is shared.

3. **Sleepnight status updated to `ordered`.** The payment receipt is unambiguous (remaining
   payment, paid in full via SPayLater). Flagged that this supersedes the still-pending
   extraction-04 proposal — whoever compiles should resolve both proposals together.

4. **Fabric swatches held at low confidence, no room/use assigned.** Zero caption, zero

   (swatch name, colour) but blocking compile until human clarifies.

## Learnings

- **Capture bot context gap**: the June 19 capture shows that when a photo is sent to the
  capture bot with an inline caption in Telegram, the caption IS saved. But follow-up reply
  messages from the user (adding more context about the same photo) are NOT captured — only the
  original message+caption is saved. This caused the 13:16 fabric swatches to land with no
  context. Architecture fix tracked separately (see user conversation 2026-06-21).
