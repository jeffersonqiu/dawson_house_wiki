## Learnings

- A single transcript can span nearly every room in the wiki — grouping by ROOM (one proposal
  per room/topic area) rather than by source document kept each proposal reviewable, but
  required cross-referencing the SAME transcript across all 5 files; each proposal cites only
  the relevant outline section(s) and action item(s) rather than the whole transcript.
- Not every discussion topic needs a task: the meeting's "Action Items" list (8 items) is the
  most reliable signal for what's actually actionable vs. exploratory discussion. The Bedroom
  section had real discussion content but zero attached action items, so it became a
  notes-only proposal with no new task IDs — avoids inflating the task list with
  un-actionable/unconfirmed chatter (consistent with the Bathroom proposal's treatment of the
  shampoo-dish-colour and lighting-colour-temperature asides).
- When a meeting's action item could plausibly belong to more than one room (e.g. the
  "lighter brown wall colour instead of green" item vs. the "green wall tile" research item —
  Kitchen vs. Bathroom proposals), don't try to force a single owner — flag the cross-reference
  in BOTH proposals' open questions and let the human/Bud Studio resolve it against the actual
  renders.
- When a new action item refines an EXISTING open task's scope (shoe storage spec → TASK-0003)
  rather than introducing new scope, propose an UPDATE to the existing task instead of a new ID
  — keeps the task list from fragmenting into near-duplicates.
