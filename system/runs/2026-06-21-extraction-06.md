## Learnings

- **Purchase invoices overturn review-queue proposals:** When a corroborating invoice arrives, it can completely supersede an extraction proposal that was only working from the tracker. This extraction run had to explicitly supersede 3 prior proposals. The Compiler must handle this gracefully — it cannot blindly compile all "approved" proposals if later-dated proposals supersede them.
- **The reno sheet remains the best single source of current item status**, but individual invoices are the authoritative source for specific prices, model numbers, and delivery dates. Always cross-reference both.
