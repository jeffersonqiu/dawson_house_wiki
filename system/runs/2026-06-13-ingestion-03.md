## Learnings

- The `/ingest` change-check workflow (manifest review → `modifiedTime >` query on known parent
  folder IDs → targeted re-pull) is fast and effective — only 4 `search_files` calls were needed
  to cover the entire previously-pulled tree, and it correctly surfaced both a file-level change
  (Reno Sheet edit) and a structural change (new subfolder).
- Spreadsheet trackers like the Reno Sheet are "live" documents that get edited after the
  initial pull — they're the highest-value re-check target since they aggregate decisions across
  many vendors/rooms. Worth re-checking this file specifically on a regular cadence even if
  nothing else changed.
- When a tracker row changes product+price for an item already compiled, don't silently
  overwrite — the diff should flow through Extractor → review_queue → Compiler like any other
  fact change, preserving the audit trail (this run intentionally stopped at "raw" and flagged
  follow-ups rather than touching `Dawson's wiki/wiki/` directly, since Ingestion may not write
  there).
